# CopyPhoto

Первая очередь одноразовой программы для выделения бумажных фотографий,
разложенных на однотонной подложке и снятых на iPhone параллельно столу.

Текущая версия реализует графическое управление, обнаружение, кадрирование,
пакетный экспорт и структурированную диагностику:

- чтение HEIC/HEIF, JPG и PNG;
- применение EXIF-ориентации;
- обнаружение фотографий средствами OpenCV;
- выделение фотографий в полном разрешении, перспективное выравнивание краёв
  и настраиваемый поворот длинной стороной по горизонтали;
- сохранение каждой фотографии в отдельный JPEG или PNG без перезаписи;
- первичная коррекция в режимах «Без коррекции» и «Мягкая»;
- диагностические изображения с найденными и отклонёнными контурами;
- причины отклонения контуров и рекомендации по исправлению условий съёмки;
- редактирование `settings.ini` через интерфейс PyQt6;
- просмотр JPG, PNG и HEIC/HEIF во входном, выходном, итоговом и
  диагностическом каталогах;
- перенос выбранных готовых фотографий в защищённый итоговый каталог;
- безопасная очистка рабочих каталогов с перемещением изображений в Корзину;
- запуск обработки в фоновом потоке и просмотр журнала работы.

Расширенная цветокоррекция будет добавлена на следующих этапах.

## Установка

Для запуска исходного кода и сборки EXE требуется Windows и Python 3.14.
Именно на этой версии разрабатывалась и проверялась программа. Проверить
наличие Python 3.14 можно командой:

```powershell
py -3.14 --version
```

Если команда сообщает об отсутствии Python, его нужно установить, а затем
открыть новое окно PowerShell.

Скачать репозиторий, перейти в его каталог и создать отдельное виртуальное
окружение:

```powershell
git clone https://github.com/LeonidBolshakov/CopyPhoto.git
Set-Location CopyPhoto
py -3.14 -m venv .venv
```

Установить зависимости программы, тестов и сборки EXE:

```powershell
.\.venv\Scripts\python.exe -m pip install -r requirements.txt
```

Все следующие команды README выполняются из корневого каталога CopyPhoto.

## Настройка

Все операторские параметры доступны на вкладке «Настройки» графического
интерфейса. Изменения автоматически проверяются и записываются во внешний файл
`settings.ini`, в том числе при закрытии окна; комментарии файла сохраняются.
Кнопка «Восстановить по умолчанию» возвращает стандартные значения после
подтверждения. При необходимости INI можно редактировать и вручную. Он разделён
на привычные разделы:

- `[Каталоги]` — входные, готовые и отобранные итоговые фотографии;
- `[Сохранение]` — формат, имя файла и качество JPEG;
- `[Обработка]` — ориентация, режим и интенсивность коррекции.
- `[Диагностика]` — режим отладки и каталог диагностических файлов.

Перед каждым параметром находятся комментарии с его назначением и допустимыми
значениями. Строки, начинающиеся с `;` или `#`, полностью игнорируются
программой. Допустимы также комментарии после значения:

```ini
; Возможные значения: png, jpg или jpeg.
Формат = png

; Возможные значения: Без коррекции или Мягкая.
Режим коррекции = Мягкая

; Значение от 0 до 100.
Интенсивность коррекции = 25

[Диагностика]

; Возможные значения: Да или Нет.
Режим отладки = Да

; Используется только при включённом режиме отладки.
Каталог = debug
```

Относительные пути отсчитываются от каталога CopyPhoto; можно использовать и
полные пути. При повторном запуске нумерация продолжается после последнего
существующего файла, а имеющиеся результаты не перезаписываются. Качество JPEG
используется только при формате `jpeg`.

Режим «Без коррекции» не изменяет пиксели после кадрирования. Режим «Мягкая»
локально улучшает контраст яркостного канала и смешивает результат с оригиналом.
Коррекция выполняется после геометрического выравнивания и до записи JPEG или
PNG.

`album_processor/settings.py` является служебным загрузчиком: он проверяет
разделы и значения,
преобразует проценты и ответы «Да/Нет» во внутренние типы. Алгоритмические
пороги детектора, кадрирования и CLAHE остаются в `album_processor/config.py` и
не предназначены для повседневного изменения оператором.

При значении `Режим отладки = Нет` каталог диагностики не создаётся и файлы с
рамками и масками не записываются. Подробные консольные отчёты по каждому
исходному файлу, включая предупреждения и причины отклонения контуров, также
скрываются. Общие параметры запуска и итоговая статистика остаются доступными.

## Архитектура

`AlbumProcessor` из `album_processor/processor.py` управляет полным процессом,
но не содержит алгоритмов обработки изображений. Ответственность разделена
следующим образом:

| Компонент                       | Ответственность                                                                   |
|---------------------------------|-----------------------------------------------------------------------------------|
| `processor.py / AlbumProcessor` | Обход входной папки, последовательность этапов, обработка ошибок и итоговый отчёт |
| `batch.py`                      | Совместимый функциональный интерфейс пакетного запуска                            |
| `config.py`                     | Типы, проверка и внутренние значения конфигурации по умолчанию                    |
| `detector.py`                   | Поиск фотографий, оценка фона, причины отклонения контуров и предупреждения       |
| `cropper.py`                    | Полноразмерное кадрирование и геометрическое выравнивание                         |
| `enhancer.py`                   | Первичная коррекция уже вырезанной фотографии                                     |
| `image_io.py`                   | Поиск и чтение HEIC/JPEG/PNG, запись JPEG/PNG                                     |
| `image_validation.py`           | Общая проверка формы, размера и типа BGR-изображений                              |
| `naming.py`                     | Формирование имён и продолжение нумерации без перезаписи                          |
| `settings.py`                   | Чтение и проверка операторского файла `settings.ini`                              |
| `settings_editor.py`            | Безопасное сохранение значений с сохранением комментариев INI                     |
| `main.py`                       | Только консольное представление настроек и структурированного отчёта              |
| `gui.py`                        | Интерфейс PyQt6, просмотр изображений, журнал и фоновый запуск                     |
| `settings_form.ui`              | Редактируемая в Qt Designer форма операторских настроек                           |

Для программного запуска используется класс:

```python
from album_processor.processor import AlbumProcessor
from album_processor.settings import load_settings

settings = load_settings()
summary = AlbumProcessor(
    settings.detector_config,
    settings.export_config,
    cropper_config=settings.cropper_config,
    enhancer_config=settings.enhancer_config,
    diagnostics_config=settings.diagnostics_config,
).process()
```

Функция `process_input_directory()` в `album_processor/batch.py` сохранена как
совместимый функциональный интерфейс.

## Графический запуск

Из Проводника можно дважды щёлкнуть `run.bat` в корневом каталоге проекта.
Из PowerShell тот же файл запускается командой:

```powershell
.\run.bat
```

Сценарий использует `.venv\Scripts\pythonw.exe` рядом с проектом и запускает
`gui.py`. В окне доступны вкладки «Настройки», «Изображения» и «Журнал».
Параметры автоматически сохраняются после изменения. Кнопка «Запустить
обработку» окончательно проверяет их, запускает ядро в рабочем потоке и после
завершения обновляет списки файлов.

Во вкладках «Входные» и «Готовые» можно выделить одну, несколько или сразу все
фотографии и перенести их кнопкой «В итоговые…». Входные файлы получают общий
штамп времени начала операции `ВГГ-ММ-ДД-ЧЧ-ММ_`, например
`В26-07-22-14-35_IMG_001.heic`. Готовые фотографии сохраняют исходное имя, если
оно свободно в итоговом каталоге. При совпадении к исходному имени добавляется
номер версии: рядом с `photo_0004.jpg` появятся `photo_0004_2.jpg`, затем
`photo_0004_3.jpg`. Это позволяет сравнить совпавшие фотографии и решить,
являются ли они дубликатами или разными версиями. Существующие файлы не
перезаписываются. Из итогового каталога программа файлы не удаляет, поэтому
кнопки очистки на его вкладке нет.

На вкладках остальных каталогов есть кнопка «Очистить…». После подтверждения
она перемещает в Корзину только показанные JPG, JPEG, PNG и HEIC/HEIF. Сам
каталог, вложенные папки и файлы других типов не удаляются. Во время обработки
очистка и перенос заблокированы. Кнопка «Обновить список» повторно читает
содержимое соответствующего каталога с диска.

Прямой графический запуск через Python:

```powershell
.\.venv\Scripts\pythonw.exe gui.py
```

## Консольный запуск

Консольный режим сохранён для автоматизации и диагностики:

```powershell
.\run_console.bat
```

Прямой запуск через Python:

```powershell
.\.venv\Scripts\python.exe main.py
```

Программа ищет `settings.ini` в корне проекта. Поэтому исходный файл можно
запускать из любого текущего каталога: относительные пути из настроек всё равно
отсчитываются от каталога CopyPhoto.

## Сборка EXE

Для сборки используется PyInstaller. Он установлен через `requirements.txt`
вместе с остальными зависимостями. Команда сборки:

```powershell
.\build_exe.bat
```

По умолчанию сценарий использует `.venv\Scripts\python.exe`. Путь к другому
окружению можно передать первым параметром:

```powershell
.\build_exe.bat .\.build-venv\Scripts\python.exe
```

`build_exe.bat` выбирает интерпретатор и запускает `build_exe.py`. Python-сценарий
проверяет наличие `CopyPhoto.spec`, `CopyPhoto.version_info` и `settings.ini`,
вызывает PyInstaller и копирует внешний файл настроек в каталог `dist`.
`CopyPhoto.version_info` задаёт отображаемые Windows название продукта и версию
EXE; эти сведения проверяются при автоматической сборке.

Результат находится в каталоге `dist`:

- `CopyPhoto.exe` — графическая программа, собранная в один исполняемый файл;
- `settings.ini` — отдельный операторский файл, который не входит в EXE.

При запуске `CopyPhoto.exe` файл `settings.ini` ищется рядом с ним. Относительные
пути `input`, `output`, `final` и `debug` также отсчитываются от каталога EXE. Настройки
можно изменять без повторной сборки программы. Для переноса программы требуется
копировать `CopyPhoto.exe` и `settings.ini` вместе.

Запуск собранного графического приложения:

```powershell
.\dist\CopyPhoto.exe
```

Файл `CopyPhoto.spec` включает в EXE форму `settings_form.ui`, но не включает
операторский `settings.ini`. Скрипт после сборки копирует INI в `dist` как
обычный внешний файл, поэтому параметры можно менять без повторной сборки.

Если в `settings.ini` включён режим отладки, для каждого исходного изображения
в выбранном диагностическом каталоге создаются:

- `<имя>_detected.jpg` — принятые и отклонённые контуры;
- `<имя>_mask.png` — маска отделения фотографий от подложки.

На диагностическом изображении зелёные рамки обозначают найденные фотографии,
а тонкие оранжевые линии — контуры, которые детектор проверил, но отклонил.
Красная рамка и символы `!!!` означают, что для исходного изображения создано
предупреждение.

Каждая обнаруженная фотография проходит полноразмерное перспективное
выравнивание и записывается в `output` в выбранном формате. Ошибка отдельного
исходника или одной фотографии отражается в итоговой статистике и не
останавливает остальные.

Фон оценивается по однородным участкам периметра кадра. В режиме отладки
консольный отчёт выводит долю участков, вошедших в доминирующую цветовую группу,
и рассчитанный порог. Если оценка ненадёжна, предупреждение содержит не только
значение, но и рекомендацию: освободить периметр, выровнять освещение, убрать
блики или тени.

В режиме отладки для отклонённых контуров выводятся количества по причинам:

- слишком малая или слишком большая площадь;
- нулевая прямоугольная рамка;
- слишком короткая сторона;
- низкая прямоугольность;
- неподходящее соотношение сторон;
- слишком большой наклон.

Если фотографии не найдены, программа отдельно сообщает об этом и показывает
наиболее частую причину отклонения. Ошибка отдельного исходника, диагностики или
одной фотографии записывается в отчёт и не останавливает остальные файлы.

Режим отладки можно отключить после завершения настройки оборудования и
проверки качества обнаружения.

## Вспомогательная палитра LAB

Модуль `lab_color_palette.py` помогает наглядно оценить цветовые расстояния,
которые используются алгоритмами обнаружения фотографии и удаления остаточной
полосы подложки. Модуль является вспомогательным, не импортируется основной
программой и не участвует в обработке фотографий.

Запуск из каталога CopyPhoto:

```powershell
.\.venv\Scripts\python.exe -m lab_color_palette
```

За исходный цвет подложки принимается светло-серый `BGR (220, 220, 220)`.
Палитра показывает отображаемые цвета с расстояниями `14`, `18`, `32` и `80`,
а также наиболее удалённый найденный цвет. Для каждого образца указываются
заданное и фактическое расстояния, координаты BGR и LAB.

Результат сохраняется в `debug/lab_color_distances.png`.

## Автоматическая проверка и сборка

GitHub Actions выполняет проверяемую сборку Windows по сценарию
`.github/workflows/build-windows.yml`. При изменении ветки `master`, создании
запроса на включение изменений и запуске вручную процесс:

1. устанавливает Python 3.14 и зафиксированные зависимости;
2. запускает все автоматические тесты;
3. выполняет статическую проверку типов;
4. собирает однофайловый `CopyPhoto.exe`;
5. проверяет название и версию продукта в ресурсах Windows;
6. публикует EXE и внешний `settings.ini` как артефакт конкретного коммита.

Это связывает результат сборки с исходным кодом и позволяет передавать в
SignPath именно автоматически собранный файл.

## Конфиденциальность

CopyPhoto обрабатывает фотографии локально, не собирает телеметрию и не
передаёт изображения или другие сведения во внешние системы. Подробности
приведены в [политике конфиденциальности](PRIVACY.md).
