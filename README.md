# CopyPhoto / AlbumProcessor

Первая очередь одноразовой программы для выделения бумажных фотографий,
разложенных на однотонной подложке и снятых на iPhone параллельно столу.

Текущий этап реализует только шаги 2–3:

- чтение HEIC/JPG/PNG;
- применение EXIF-ориентации;
- обнаружение фотографий средствами OpenCV;
- диагностические изображения с найденными рамками и масками.

Кадрирование, устранение небольшого наклона, коррекция и интерфейс PyQt6
будут добавлены после настройки детектора на реальных снимках с подложки.

## Установка

```powershell
.\.venv\Scripts\python.exe -m pip install -r requirements.txt
```

## Настройка

Параметры детектора и их значения по умолчанию находятся в
`album_processor/config.py`. В `settings.py` задаются только каталоги,
которые создаются рядом с проектом:

- `input` — исходные HEIC/JPG/PNG;
- `debug` — рамки и бинарные маски детектора.

## Запуск

```powershell
.\.venv\Scripts\python.exe main.py
```

Для каждого исходного изображения в `debug` создаются:

- `<имя>_detected.jpg` — найденные контуры;
- `<имя>_mask.png` — маска отделения фотографий от подложки.

Фон оценивается по однородным участкам периметра кадра. Программа выводит
долю участков, вошедших в доминирующую цветовую группу. Если покрытие слишком
мало или рассчитанный порог аномален, в консоли появляется предупреждение,
а диагностическое изображение выделяется красной рамкой с символами `!!!`.

Диагностический вывод нужен только для разработки и будет отключён в
итоговой версии.
